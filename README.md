# BuffBites

AI-powered meal combo suggester for CU Boulder dining halls. Give it a dining hall and date, get back 9 combos (3 Breakfast, 3 Lunch, 3 Dinner) generated by Claude.

---

## Stack

- **Backend:** Python, FastAPI, Uvicorn
- **Validation:** Pydantic v2 (`backend/pydantic_models/`)
- **AI:** Anthropic Claude (`claude-haiku-4-5`)
- **Data:** Pre-scraped JSON menus in `scraping_scripts/data/`

---

## Run

```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload --port 3001
```

Requires a `.env` file in `backend/`:
```
ANTHROPIC_API_KEY=...
```

Interactive API docs available at `http://localhost:3001/docs`.

---

## API

`GET /api/combos/generate?dining=<hall>&date=<YYYY-MM-DD>`

Valid `dining` values:

| Value | Dining Hall |
|---|---|
| `alley` | The Alley |
| `c4c` | Center for Community (C4C) |
| `libby` | Libby Dining |
| `sewall` | Sewall Dining |
| `village_center` | Village Center Dining |

`date` defaults to today if omitted.

**Examples:**
```bash
# C4C
curl "http://localhost:3001/api/combos/generate?dining=c4c&date=2026-03-17"

# The Alley
curl "http://localhost:3001/api/combos/generate?dining=alley&date=2026-03-17"

# Libby
curl "http://localhost:3001/api/combos/generate?dining=libby&date=2026-03-17"

# Sewall
curl "http://localhost:3001/api/combos/generate?dining=sewall&date=2026-03-17"

# Village Center
curl "http://localhost:3001/api/combos/generate?dining=village_center&date=2026-03-17"
```

**Response shape:**
```json
{
  "dining_location": "Center for Community (C4C)",
  "date": "2026-03-17",
  "day_of_week": "Tuesday",
  "combos": {
    "Breakfast": [
      {
        "title": "Combo Name",
        "dishes": [
          { "name": "Item 1", "station": "Grill" },
          { "name": "Item 2", "station": "Salad Bar" }
        ],
        "description": "Why this combo works",
        "approximate_calories": 600,
        "tags": ["high-protein", "vegetarian"]
      }
    ],
    "Lunch":  ["...3 combos, same shape..."],
    "Dinner": ["...3 combos, same shape..."]
  }
}
```

See `API_DOCS.md` for full details.
