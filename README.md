# BuffBites

AI-powered meal combo suggester for CU Boulder dining halls. Give it a dining hall and a date, get back **9 curated combos** вЂ” 3 for Breakfast, 3 for Lunch, and 3 for Dinner (including a dessert specialty combo) вЂ” generated by Claude and verified against live scraped menu data.

---

## How It Works

1. **Scraping** вЂ” Python scripts pull menus from Nutrislice for each CU Boulder dining hall and store them as JSON in `scraping_scripts/data/`.
2. **API Request** вЂ” Client calls `GET /api/combos/generate?dining=c4c&date=2026-03-17`.
3. **Menu Lookup** вЂ” FastAPI loads the scraped JSON for that dining hall and finds the day's menu.
4. **Station Classification** вЂ” Every station is classified by keyword into `breakfast`, `lunch`, `dinner`, `dessert`, or `excluded`. Utility stations (Condiments, Salad Dressings, Beverage, Sides, Bread, etc.) are excluded entirely. Individual condiment/sauce items are filtered out at the item level as a second pass.
5. **AI Generation** вЂ” Classified items are sent to `claude-haiku-4-5` in separate pools per meal period. Claude generates 9 combos (3 per period). The 3rd Dinner combo is always a dessert specialty вЂ” if ice cream or soft serve is available, Claude pairs it with cookies, brownies, cake, or fruit. If no dessert items exist, it falls back to a light plant-based dinner combo.
6. **Pydantic Validation** вЂ” The response is validated against strict Pydantic v2 models: structure, combo count, and dish count (2вЂ“6 per combo) are all enforced.
7. **Dish Verification** вЂ” Every dish Claude returned is cross-checked against the scraped menu. Hallucinated dishes and wrong station labels are logged as warnings to stderr. The response is still returned.
8. **Response** вЂ” Validated `ComboResponse` JSON is returned to the client.

## Setup & Run

### 1. Create and activate a virtual environment

```bash
cd backend
python3 -m venv venv
source venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment

Create a `.env` file inside `backend/`:

```env
ANTHROPIC_API_KEY=your_key_here
```

### 4. Start the server

```bash
uvicorn main:app --reload --port 3001
```

The API will be available at `http://localhost:3001`.
Interactive Swagger docs at `http://localhost:3001/docs`.

## API Quick Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Health check |
| `GET` | `/api/combos/generate` | Generate 9 meal combos |

**Required query param:** `dining` вЂ” one of `alley`, `c4c`, `libby`, `sewall`, `village_center`
**Optional query param:** `date` вЂ” `YYYY-MM-DD`, defaults to today

```bash
curl "http://localhost:3001/api/combos/generate?dining=c4c&date=2026-03-17"
```

See [`API_DOCS.md`](./API_DOCS.md) for full request/response details.
